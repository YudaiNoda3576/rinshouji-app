## 2. 不整合・未接続項目

### 2.1 「直近の予定」列（唯一の未接続列）

- `TableView.tsx:46` で `<span className="dim">—</span>` の固定表示。`HouseholdRow`・API レスポンス・DB のいずれにも対応データが存在しない。
- ただしこれは設計どおりの暫定状態: [画面項目定義.md](画面項目定義.md) §1.1 で「スケジュール機能由来（本設計時点では未接続）。フェーズ2でスケジュール機能と接続するまでは空表示または『—』固定でも可」と定義済み。
- **結論: 仕様上は整合（意図的な未接続）。実装上はスケジュール機能接続時に列のデータバインドが必要**。

### 2.2 検索専用項目（列には出ないが SQL の検索対象）

検索ボックス `q` の ILIKE 対象（`households.ts:55-60`）:

| 検索対象 | DB カラム | 一覧列 |

| 家名 | `households.family_name` | あり |
| 戸主名 | `household_members.name` | あり |
| フリガナ（戸主） | `household_members.name_kana` | なし（設計どおり検索専用） |
| 住所1 | `households.address_1` | なし（同上） |

設計（[画面項目定義.md](画面項目定義.md) §1.1「検索対象を満たすための非表示項目」）と実装は一致している。

## 4. 拡張案: 一覧への列追加（将来実装）

一覧でも確認したい項目として以下を追加候補とする。**表示方式は「列の表示/非表示切替」**とし、既定の列構成は現行 7 列のまま変えない（画面崩れ・SP 方針への影響を避けるため）。

### 4.1 追加候補列

| 追加列 | DB カラム | 表示形式案 |
| --- | --- | --- |
| 住所 | `postal_code` + `address_1` + `address_2` | 「〒486-XXXX 春日井市…」の結合1列（詳細の `formatPostal` + 連結表示を流用） |
| 位牌区分 | `ihai_status` | バッジ（あり=青/なし=グレー、詳細パネルと同配色） |
| 備考 | `note` | テキスト（1行省略表示、ホバーで全文） |

### 4.2 表示/非表示切替の方針

- テーブル右上に列選択メニュー（チェックボックスのドロップダウン）を設置。
- 既定 ON: 現行 7 列。既定 OFF: 住所・位牌区分・備考。
- 選択状態は `localStorage` に永続化（画面再訪時も維持）。
- SP（`sp.css` の 640px ブレークポイント）では現行どおり家名のみ表示の方針を維持し、追加列も非表示対象とする。

### 4.3 必要になる変更の概要（実装時の作業範囲）

1. **API**: `GET /api/households` のレスポンスに `postalCode` / `address1` / `address2` / `ihaiStatus` / `note` を追加（`households.ts` の SELECT 句とマッピング）。
2. **フロント型**: `HouseholdRow`（`types.ts`）に同フィールドを追加。
3. **UI**: `TableView.tsx` の列を配列ベースの列定義（key・ヘッダー・セル描画関数・既定表示フラグ）にリファクタし、列選択メニューと `localStorage` 永続化を実装。
