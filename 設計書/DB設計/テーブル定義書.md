# テーブル定義書

林昌寺 檀信徒管理システムの正規化データモデルのテーブル定義。全体像・設計判断は [データモデル設計.md](./データモデル設計.md)、列単位の移行方針は [データ移行設計.md](./データ移行設計.md) を参照。

- DBMS: PostgreSQL
- 全テーブルの主キーは surrogate key（`BIGSERIAL`）。
- **CSV 原文はステージングテーブル `import_records` に一本化して保持する。** 本番テーブルに残すのは「画面表示に使うカラムだけ」で、取込元 CSV 行への参照は `import_record_id` FK で行う（判定基準の詳細は [データモデル設計.md](./データモデル設計.md) 「6.4 ステージング分離方針」）。
- enum は `VARCHAR + CHECK` 制約で表現する。
- 全テーブルに `created_at` / `updated_at`（`TIMESTAMP` / 既定 `CURRENT_TIMESTAMP`）を付与する（追記専用の `import_records` のみ例外で `imported_at` のみ）。

## テーブル一覧

| 論理名 | 物理名 | 概要 |
| --- | --- | --- |
| 取込レコード | `import_records` | CSV 原文のステージング。1 CSV 行 = 1 レコード、全列を JSONB に保持 |
| 地区 | `districts` | 地区区分1→区分2 の階層を自己参照で表現するマスタ |
| 世帯 | `households` | 家（戸）単位の台帳。連絡先・行事設定・関係区分を内包し soft delete を持つ |
| 世帯構成員 | `household_members` | 世帯に属する存命人物（現戸主・前戸主・家族）|
| 故人 | `deceased_persons` | 過去帳の故人。世帯 FK は NULL 可 |
| 墓地区画 | `cemetery_plots` | 世帯が保有する墓地区画（1 世帯 : 多区画）|
| 納骨堂区画 | `columbarium_units` | 世帯が利用する納骨堂の区画・一時預かり |

## 0. 取込レコード（import_records）

### テーブル概要
| 項目 | 内容 |
| --- | --- |
| テーブル論理名 | 取込レコード |
| テーブル物理名 | `import_records` |
| 概要 | 移行元 CSV の原文ステージング。1 CSV 行 = 1 レコードとして全列を `raw_data` JSONB に列名キーで丸ごと保持する。移行検証・原文参照・意味未確定データの退避を一元的に担う |

### カラム定義
| No. | 論理名 | 物理名 | データ型 | 桁数 | NULL許可 | 主キー | 外部キー | 初期値 | 備考 |
| --- | --- | --- | --- | --- | --- | --- | --- | --- | --- |
| 1 | ID | `id` | BIGSERIAL |  | NO | YES |  |  | surrogate key |
| 2 | 取込元ファイル | `source_file` | VARCHAR | 20 | NO |  |  |  | CHECK (`source_file IN ('檀信徒名簿','過去帳')`) |
| 3 | 行番号 | `row_number` | INTEGER |  | NO |  |  |  | CSV のデータ行番号（ヘッダ除く 1 始まり）|
| 4 | 原文データ | `raw_data` | JSONB |  | NO |  |  |  | 全列を列名キーで丸ごと保持（無名列は「無名列」等のキーで格納）|
| 5 | 取込日時 | `imported_at` | TIMESTAMP |  | NO |  |  | CURRENT_TIMESTAMP | 追記専用のため created_at/updated_at は持たない |

### 制約・インデックス
- CHECK: `source_file IN ('檀信徒名簿','過去帳')`。
- **UNIQUE: `(source_file, row_number)`**（取込元行の一意化。再取込時の重複防止）。

## 1. 地区（districts）

### テーブル概要
| 項目 | 内容 |
| --- | --- |
| テーブル論理名 | 地区 |
| テーブル物理名 | `districts` |
| 概要 | 名簿の地区区分1・区分2 を自己参照階層で正規化するマスタ |

### カラム定義
| No. | 論理名 | 物理名 | データ型 | 桁数 | NULL許可 | 主キー | 外部キー | 初期値 | 備考 |
| --- | --- | --- | --- | --- | --- | --- | --- | --- | --- |
| 1 | ID | `id` | BIGSERIAL |  | NO | YES |  |  | surrogate key |
| 2 | 親地区ID | `parent_id` | BIGINT |  | YES |  | `districts.id` |  | 区分2 の場合に区分1 を指す。区分1 は NULL |
| 3 | 地区名 | `name` | VARCHAR | 50 | NO |  |  |  | 正規化後の地区名 |
| 4 | 階層 | `level` | SMALLINT |  | NO |  |  |  | CHECK (`level IN (1,2)`)。1=区分1 / 2=区分2 |
| 5 | 作成日時 | `created_at` | TIMESTAMP |  | NO |  |  | CURRENT_TIMESTAMP |  |
| 6 | 更新日時 | `updated_at` | TIMESTAMP |  | NO |  |  | CURRENT_TIMESTAMP |  |

### 制約・インデックス
- CHECK: `level IN (1, 2)`。`level=2` のとき `parent_id IS NOT NULL`、`level=1` のとき `parent_id IS NULL` をアプリ層または CHECK で担保。
- INDEX: `parent_id`（階層辿り用）、`name`。
- UNIQUE: `(parent_id, name)` 推奨（同一親配下での地区名重複防止）。

| 10 | 電話 | `phone` | VARCHAR | 20 | YES |  |  |  | 固定電話。???・*** はクレンジング |
| 11 | 携帯電話 | `mobile_phone` | VARCHAR | 50 | YES |  |  |  | 名前prefix・複数番号ありのため桁広め |
| 12 | 般若 | `hannya_service` | VARCHAR | 10 | YES |  |  |  | CHECK。組/郵送/なし |

| 15 | 月参り日 | `monthly_service_day` | VARCHAR | 20 | YES |  |  |  | 月経。数値/複雑値を原文保持 |
| 16 | 地蔵 | `jizo_flag` | BOOLEAN |  | NO |  |  | `false` | 名簿「地蔵」= true |
| 17 | 位牌区分 | `ihai_status` | VARCHAR | 10 | YES |  |  |  | CHECK。あり/なし |
| 18 | 備考 | `note` | TEXT |  | YES |  |  |  | 世帯全体の備考 |
| 19 | 作成日時 | `created_at` | TIMESTAMP |  | NO |  |  | CURRENT_TIMESTAMP |  |
| 20 | 更新日時 | `updated_at` | TIMESTAMP |  | NO |  |  | CURRENT_TIMESTAMP |  |

### 制約・インデックス
- CHECK: `status IN ('active','inactive','deleted')` / `relation_type IN ('檀家','信徒','檀徒','他寺','その他')` / `hannya_service IN ('組','郵送','なし')` / `sejiki_service IN ('組','郵送','なし','不要')` / `ihai_status IN ('あり','なし')`（各 NULL 許容）。
- INDEX: `family_name`（家名検索）、`district_id`、`status`。
- 検索補助: 構成員の氏名・カナ検索は `household_members` 側のインデックスで対応。旧家番での突合は `import_record_id` → `import_records.raw_data` 経由で行う。

## 3. 世帯構成員（household_members）

### テーブル概要
| 項目 | 内容 |
| --- | --- |
| テーブル論理名 | 世帯構成員 |
| テーブル物理名 | `household_members` |
| 概要 | 世帯に属する存命人物。役割（現戸主・前戸主・家族）で区別する |

### カラム定義
| No. | 論理名 | 物理名 | データ型 | 桁数 | NULL許可 | 主キー | 外部キー | 初期値 | 備考 |
| --- | --- | --- | --- | --- | --- | --- | --- | --- | --- |
| 1 | ID | `id` | BIGSERIAL |  | NO | YES |  |  | surrogate key |
| 2 | 世帯ID | `household_id` | BIGINT |  | NO |  | `households.id` |  |  |
| 3 | 役割 | `member_role` | VARCHAR | 20 | NO |  |  |  | CHECK。head/former_head/family |
| 4 | 氏名 | `name` | VARCHAR | 100 | NO |  |  |  | **姓・名に分割せず原文保持**（全角スペース含む。名のみレコードあり）|
| 5 | フリガナ | `name_kana` | VARCHAR | 100 | YES |  |  |  | 現戸主のみ。半角カナ→全角カナへ正規化 |
| 6 | 継承順 | `succession_order` | SMALLINT |  | YES |  |  |  | 前戸主の順序（1〜4）。他役割は NULL |
| 7 | 備考 | `note` | VARCHAR | 255 | YES |  |  |  |  |
| 8 | 作成日時 | `created_at` | TIMESTAMP |  | NO |  |  | CURRENT_TIMESTAMP |  |
| 9 | 更新日時 | `updated_at` | TIMESTAMP |  | NO |  |  | CURRENT_TIMESTAMP |  |

### 制約・インデックス
- CHECK: `member_role IN ('head','former_head','family')`。
- **部分 UNIQUE インデックス**: `CREATE UNIQUE INDEX ON household_members (household_id) WHERE member_role = 'head';`（世帯あたり現戸主 1 件を担保）。
- INDEX: `household_id`、`name`、`name_kana`（氏名・カナ検索。家名検索は `households.family_name` が担う）。
